# PackCheck — 旅遊行李打包清單

**PackCheck** 是一款輕量、無需安裝的行李管理工具，協助旅行者把每趟旅程的打包需求整理成清楚的清單，並分別追蹤出發與回程的確認狀態。

---

## 功能特色

- **多旅程管理** — 可建立多個旅程（例如「日本 2026」、「歐洲蜜月」），每個旅程擁有獨立的行李清單。
- **出發 / 回程雙向追蹤** — 每件物品都有獨立的「出發確認」與「回程確認」核取方塊，避免回程遺落行李。
- **物品數量管理** — 可為每件物品指定數量，並支援行內編輯（直接修改名稱與數量）。
- **本地儲存持久化** — 所有資料儲存於瀏覽器 `localStorage`，關閉後重開仍保留；無需帳號、無需伺服器。
- **Hash 路由 SPA** — 以 `#trips` / `#trip/:id` 實作單頁應用導覽，支援瀏覽器上一頁 / 下一頁。
- **離線可用** — 純靜態頁面，無外部 API 依賴，下載後即可在本機直接開啟使用。

## 技術架構

| 項目     | 說明                                                    |
| -------- | ------------------------------------------------------- |
| 語言     | 原生 HTML / CSS / JavaScript（無框架）                  |
| 狀態管理 | 全域 `state` 物件，每次變動後同步寫入 `localStorage`    |
| 路由     | Hash-based SPA（`hashchange` 事件驅動）                 |
| 字型     | 微軟正黑體（Microsoft JhengHei）、PingFang TC           |
| 樣式     | 純 CSS，CSS 自訂屬性（Custom Properties）管理設計 token |
| 資安     | 所有使用者輸入透過 `esc()` 函式進行 HTML 跳脫，防止 XSS |

## 快速開始

不需要安裝任何套件或啟動伺服器，直接用瀏覽器開啟即可：

```
直接在瀏覽器開啟 index.html
```

或透過任意靜態伺服器（例如 VS Code Live Server、`npx serve`）執行：

```bash
npx serve .
```

## 使用方式

1. **新增旅程** — 在首頁輸入旅程名稱，點擊「+ 新增旅程」。
2. **進入旅程** — 點擊旅程卡片進入行李清單頁面。
3. **新增物品** — 輸入物品名稱與數量，點擊「+ 新增物品」。
4. **確認出發** — 打包完成後勾選「出發確認」。
5. **確認回程** — 離開住宿前勾選「回程確認」，確保不遺落任何物品。
6. **編輯 / 刪除** — 可對個別物品進行行內編輯或刪除；旅程可從首頁刪除。

## 資料格式

資料以 JSON 儲存於 `localStorage`，鍵名為 `packcheck_data`，格式如下：

```json
{
  "trips": [
    {
      "id": "1748000000000-1234567",
      "name": "11月日本土浦煙火行",
      "createdAt": "2026-05-31T00:00:00.000Z",
      "items": [
        {
          "id": "1748000000001-9876543",
          "name": "護照",
          "qty": 1,
          "departureChecked": true,
          "returnChecked": false
        }
      ]
    }
  ]
}
```
